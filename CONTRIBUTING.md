# Contributing

Contributions are welcome if they keep the project simple, readable, and safe for ERP use.

## Contribution Rules

Please follow these points:

- Use clear file names.
- Keep examples simple.
- Do not add private company data.
- Do not add real customer, bank, invoice, tax, or accounting records.
- Do not add passwords, API keys, connection strings, or server information.
- SQL report examples should be SELECT-only.
- Explain the business purpose of the change.

## SQL Examples

For SQL report examples:

- Prefer readable SELECT queries.
- Avoid unnecessary nested queries.
- Use aliases clearly.
- Do not include DELETE, UPDATE, INSERT, TRUNCATE, or direct accounting correction scripts.
- Add comments only where they are useful.

## PHP Examples

For PHP examples:

- Keep functions small.
- Use clear variable names.
- Avoid framework-specific code unless needed.
- Do not include real database credentials.

## Pull Request Notes

A good pull request should explain:

1. What problem it solves
2. Which module it affects
3. How it can be tested
4. Any business risk
