# Project Overview

DuruNex ERP Toolkit is a collection of practical ERP development notes, examples, and helper code.

The project is designed around common ERP needs:

- Sales
- Purchasing
- Inventory
- Finance
- Banking
- Approval workflows
- Authorization
- Reporting

The aim is not to create a large and complicated framework. The aim is to keep useful ERP development patterns in one clean place.

## Main Idea

ERP projects usually fail because business rules are not clear enough.

For example:

- Which amount should be used: net amount, VAT included amount, or discounted amount?
- Which date should be used: document date, due date, posting date, or delivery date?
- Which user is allowed to approve a document?
- Which warehouse movement should affect stock?
- Which report should show open records and which report should show posted records?

This repository focuses on these practical questions.

## Scope

This repository includes:

- SQL report examples
- ERP workflow documentation
- PHP helper examples
- Authorization examples
- Approval examples
- Project roadmap notes

This repository does not include:

- Real production ERP source code
- Private database access information
- Real accounting entries
- Real customer data
- Commercial project secrets
